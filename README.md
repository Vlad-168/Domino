# 🁫 Domino

**Семейное PWA для геймификации бытовых дел в паре.** Баллы, личное соревнование,
общие награды и автоматические недельные отчёты — фрейм «мы вместе против бардака»
вместо «я делаю больше тебя».

Реализовано по [техническому заданию](#соответствие-тз) v1.0.

## ✨ Возможности

- **Главная** — живой счёт пары, личные таргеты (с гандикапом), прогресс общей
  недельной цели, стрики, отметка дела в 3 тапа с конфетти.
- **Каталог дел** — CRUD: название, баллы 1–10, категория, частота, множитель
  «никто не хочет это делать» (×2), архивация без потери истории.
- **Лог / История** — лента отметок по дням, мгновенное начисление, отмена записи.
- **Награды** — каталог совместных наград, разблокировка и погашение из
  командного или личного пула баллов.
- **Отчёты** — недельный снапшот (итоги, победитель с учётом справедливости,
  общая цель, MVP-дело, тренд, стрики, разбивка по делам), сезонный зачёт W–L,
  кнопка «Закрыть неделю» (MVP) + серверный авточекпоинт (v1).
- **Достижения** — бейджи: первое дело, стрик 7 дней, 100 за неделю, командная
  цель, полуночник, чемпион сезона.
- **Гандикап** — пресеты соотношения личных целей для справедливости.
- **PWA** — установка на экран «Домой», офлайн через Service Worker, манифест
  `display: standalone`, safe-area для выреза / Dynamic Island, онбординг-экран
  с инструкцией установки на iOS и запросом разрешения на уведомления.

## 🛠 Стек

React 18 · TypeScript · Vite · Zustand (persist) · Framer Motion ·
vite-plugin-pwa (Workbox) · Vitest + Testing Library.

Серверная часть (ТЗ §5–7) — Firebase: Firestore (real-time sync двух устройств),
Auth, Hosting, Cloud Functions + Cloud Scheduler (авточекпоинт), FCM (пуши).
См. `server/functions/index.js`, `firestore.rules`, `firebase.json`.

> Клиент полностью функционален локально (Zustand + localStorage как кэш).
> Для синхронизации двух устройств подключите Firebase-проект и задеплойте
> правила и функции — облако становится единственным источником правды (ТЗ §5).

## 🔄 Синхронизация двух телефонов

Облако — Firebase Firestore, конфиг вводится **в самом приложении** (Профиль →
Синхронизация), без пересборки и секретов в репозитории. Настройка один раз:

1. Создайте бесплатный проект на <https://console.firebase.google.com> (тариф Spark).
2. **Build → Firestore Database → Create database** (можно production mode).
3. **Build → Authentication → Sign-in method → Anonymous → Enable**.
4. **Project settings → General → Your apps → Web app** → скопируйте объект
   `firebaseConfig`.
5. Опубликуйте правила доступа из `firestore.rules`
   (`firebase deploy --only firestore:rules` или вставьте в консоли Firestore → Rules).
6. В приложении на **обоих** телефонах: Профиль → Синхронизация → вставьте тот же
   `firebaseConfig` и введите **один общий код семьи** → «Подключить».

Готово: отметки, баллы, награды и недельные отчёты летят на оба устройства в
реальном времени через Firestore-листенер. Облако — единственный источник правды
(ТЗ §5); локально только кэш. На iOS подписка восстанавливается при каждом запуске.
